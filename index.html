Master-Detail

This is a best practice implementation of the master detail design pattern in UI5.
Choose one of the access points below to launch the app.

Standalone

These files can be used to run the application as a standalone application in the SAP Web IDE or after building and serving with bower/grunt (expects a UI5 installation available at the server root, so this will not work without this build and the grunt-based server).
  • testService.html - start the app with mock data
  • testFLP.html - start the app in the SAP Fiori launchpad sandbox
  • testFLPService.html - start the app in the SAP Fiori launchpad sandbox with mock data
  • production.html - start the app in production mode
  • test/unit/unitTests.qunit.html - run all unit tests
  • test/integration/opaTests.qunit.html - run all integration tests

Demokit/SDK

These files can be used to run the application from within the Demokit/SDK (using the UI5 version shipping with the SDK).
  • demokit.html - start the app with mock data in demokit context
  • demokitFLP.html - start the app in the SAP Fiori launchpad sandbox (requires ushell library)
  • test/unit/unitTestsDemokit.qunit.html - run all unit tests in demokit context
  • test/integration/opaTestsDemokit.qunit.html - run all integration tests in demokit context

For more documentation please read the UI5 Developer Guide.
